# djinni-parser

Особистий скрипт для збору публічних вакансій Frontend (React/JavaScript/TypeScript)
з [Djinni.co](https://djinni.co) у CSV — для власного пошуку роботи. Не для
комерційного використання чи перепродажу даних.

## Дотримання правил сайту

- Програмно перевіряє `djinni.co/robots.txt` перед кожним запитом
  (`urllib.robotparser`), а не покладається на захардкожений список шляхів.
- Ходить лише на `/jobs/<id>-<slug>/` та `/jobs/?...` (пошук/пагінація).
- Випадкова затримка 2-4с між запитами, без паралельних запитів, звичайний
  User-Agent браузера, без логіну/авторизації.
- Зупиняється одразу, якщо сайт сигналізує блокування (HTTP 429/403/503,
  CAPTCHA-сторінка) — не намагається "пробитись" повторними спробами.

**robots.txt — це технічна чемність до сканерів, а не заміна Terms of Service.**
Перед активним використанням варто самостійно ознайомитись з ToS Djinni.

## Встановлення

```
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
```

## Використання

```
python djinni_parser.py --dry-run              # лише порахувати вакансії, нічого не зберігати
python djinni_parser.py                        # повний збір у vacancies_frontend.csv
python djinni_parser.py --max-pages 5           # обмежити глибину пагінації на ключове слово
python djinni_parser.py --output my.csv         # інший файл виводу
```

Повторний запуск дозаписує лише нові вакансії — вже наявні в CSV (за `url`)
пропускаються, тож перерваний прогін безпечно продовжити повторним запуском
тієї самої команди.

## Що саме збирається

Дані витягуються переважно зі структурованого блоку `<script type="application/ld+json">`
(schema.org `JobPosting`), який Djinni вбудовує в кожну сторінку вакансії — це
надійніше за парсинг HTML-верстки, яка може змінюватись. З цього ж блоку
береться `title`, `company`, `description`, `posted_date`, `salary_from/to`
(коли роботодавець його вказав) та `experienceRequirements`.

Поля, яких немає у структурованих даних, розпізнаються евристично з тексту
опису (регулярки, ключові слова) — і мають відповідні обмеження:

- **`salary_from`/`salary_to`** — здебільшого будуть порожні: більшість
  роботодавців на Djinni не вказують точну вилку.
- **`english_level`** — вільний текст ("Strong English", "англійська B2",
  "Upper-Intermediate") без єдиного словника, тож можливі пропуски на
  нетипових формулюваннях.
- **`experience_level`** — спершу шукає явне слово (junior/middle/senior/
  trainee) у назві й описі, і лише якщо не знайдено — оцінює за роками
  досвіду зі структурованих даних.
- **`work_format`** — `remote`, якщо Djinni сам позначив вакансію як
  дистанційну; інакше визначається з тексту опису (може розходитись, якщо
  опис суперечить офіційній позначці Djinni).

## Обмеження

CSS-класи/структура HTML сторінки можуть з часом змінитись — якщо парсинг
почне повертати багато порожніх полів, варто звірити актуальну розмітку
сторінки вакансії з кодом у `djinni_parser.py`.
